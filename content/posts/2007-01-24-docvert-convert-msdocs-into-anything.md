![Docvert logo](http://static.blogo.it/ossblog/docvert.png)

> This [web service](http://en.wikipedia.org/wiki/Web_Services) software takes multiple word processor files (typically .doc) and converts them to [OpenDocument](http://en.wikipedia.org/wiki/OpenDocument) and [HTML](http://en.wikipedia.org/wiki/HTML).
>> Web Service receives .doc file and converts it to a Oasis OpenDocument 1.0 which can then be converted to HTML, RSS, or any XML format.
>> The resulting OpenDocument is then optionally converted to HTML or any XML. This is done with [XML Pipelines](http://holloway.co.nz/docvert/faq.html#xml-pipelines), an approach that supports XSLT, breaking up content over headings or sections, and saving those results to multiple files (e.g., chapter1.html, chapter2.html…).
>> The result is returned in a .zip file. 



[Official Web Site](http://holloway.co.nz/docvert/index.html).
The software is a PHP Script that expose a WS and a Web Interface, that help and symplify to reach its aim.

Source, [OSSBlog](http://www.ossblog.it/post/1850/conversione-in-odf-tramite-web-service).
